# Step 2, Draft MOC

## MANDATORY EXECUTION RULES:

- 🛑 NEVER list a note without briefly commenting it (a MOC explains, it does not just link)
- ✅ ALWAYS use wiki-links `[[Exact Title]]` (resolved against the `name:` or `title:` frontmatter)
- ✅ ALWAYS respect the strict frontmatter `type: note` + `subtype: moc` (or `subtype: index` if `-t index`)
- 📋 YOU ARE a navigation-hub writer: structure AND annotate

## EXECUTION SEQUENCE:

### 1. Load the canonical template

Read `templates/atlas-moc-template.md` as the base.

For specific patterns (by topic type), consult `.claude/skills/moc-new/references/moc-patterns.md`.

### 2. Base skeleton

```markdown
---
title: MOC, {Capitalized topic}
type: note
subtype: {moc | index}
schema_version: 1
date: {YYYY-MM-DD}
status: active
topics: [{topic}, {related_1}, {related_2}]
---

# MOC, {Capitalized topic}

_Map of Content, the entry point to navigate the subject **{topic}** across the workspace._

## Overview

{2-3 sentences that frame the subject, its strategic value for the organization, and why it deserves a MOC.}

## Linked entities

{Wiki-links to the relevant atlas/topics/, atlas/services/, atlas/people/}

{Following sections depending on {structure_mode}}

## Distilled knowledge

{Links to knowledge/}

## Decisions and learnings

{Links to decisions/notes over time}

## External sources

{URLs, references, books}

## Relations

- about [[{Main topic}]]
- contains [[{sub-topic 1}]]
- contains [[{sub-topic 2}]]
```

#### Mode "AI decides"

Choose based on the observed density:

- **Several clients, few patterns** -> by client/partner
- **Few clients, many types** -> by note type
- **Structured technical topic** (accessibility, technical SEO, etc.) -> by sub-theme
- **Mix** -> hybrid: by client at the top, by sub-theme at the bottom

### 4. "Ongoing projects" section (dataview optional)

If Obsidian is used and `{map_type}` = MOC:

```dataview

FROM "sources"
WHERE contains(topics, "{topic-slug}") AND status = "active"
SORT last_updated DESC
LIMIT 20
```

Double it with the static version generated by `generate_indexes.py` if present:


### 6. "Linked entities" section (atlas/)

Depending on what exists in `atlas/`:

```markdown

- Service, [[Accessibility Audit]] (`atlas/services/accessibility-audit.md`)
- Key people, [[Jane Doe]], [[John Smith]]
- Adjacent topics, [[Accessibility]], [[Web Standards]]
```

### 7. Bottom-of-page Relations (Basic Memory)

```markdown
## Relations

- about [[{Main topic}]]
- contains [[{sub-topic 1}]]
- contains [[{sub-topic 2}]]
- related_to [[{adjacent topic}]]
```

### 8. Self-review

Before presenting to the user, verify:

```
[ ] Complete frontmatter (title, type, subtype, status, date, topics)
[ ] subtype: moc (or index if -t index)
[ ] Overview present and > 1 sentence
[ ] Each section has at least one referenced note (otherwise remove the section)
[ ] Each wiki-link has an annotation (1 contextual sentence)
[ ] No duplicate links across sections
[ ] Relations section consistent with topics:
[ ] Naming convention: atlas/maps/MOC-{slug}.md
```

If a check fails, fix it before step-03.

Store the content in `{draft_content}`.
