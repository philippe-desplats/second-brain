# Demo Content Manifest

Demo content created with the boilerplate. It exists so a new user can SEE the second brain working (a fictional client, project, emails, a meeting, a distilled playbook, a person, and a topic, all schema-correct and cross-linked) before filling the workspace with their own material.

`/sb-init` offers to remove it during initialization. Deleting every path listed here removes all traces of the demo. All content is fictional (client "Globex", contact "Jane Doe", domain `globex-example.com`).

## Removal roots

- `sources/clients/globex/` (client context, project brief, emails, meeting)
- `atlas/people/jane-doe.md`
